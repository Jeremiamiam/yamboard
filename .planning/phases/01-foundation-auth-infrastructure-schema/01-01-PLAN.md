**dashboard/src/lib/supabase/client.ts** (browser client — singleton, pour les Client Components) :

```typescript
import { createBrowserClient } from '@supabase/ssr'

export function createClient() {
  return createBrowserClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )

```

Règle critique : NE PAS appeler createBrowserClient à l'intérieur d'un composant React — cela crée plusieurs instances GoTrueClient et provoque des race conditions. Ce fichier exporte une fonction factory utilisée une seule fois au niveau module.

**dashboard/src/lib/supabase/server.ts** (server client — per-request, pour Server Components / Server Actions / Route Handlers) :

```typescript
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'

export async function createClient() {
  const cookieStore = await cookies()

  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return cookieStore.getAll()
        },
        setAll(cookiesToSet) {
          try {
            cookiesToSet.forEach(({ name, value, options }) =>
              cookieStore.set(name, value, options)
            )
          } catch {
            // Server Component : les cookies sont en lecture seule ici.
            // Le middleware gère le refresh — ignorer cette erreur est correct.
          }
        },
      },
    }
  )
}
```

Règles critiques :
- Utiliser UNIQUEMENT getAll/setAll — jamais get/set/remove individuels (pattern @supabase/ssr 0.x)
- La fonction doit être async (await cookies())
- NE PAS créer un singleton module-level — créer per-request
