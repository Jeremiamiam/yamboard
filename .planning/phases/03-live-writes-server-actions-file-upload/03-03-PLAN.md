<!-- Schema SQL de la table documents -->
```sql
CREATE TABLE documents (
  id                  UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  client_id           UUID REFERENCES clients(id) ON DELETE CASCADE NOT NULL,
  project_id          UUID REFERENCES projects(id) ON DELETE SET NULL,
  name                TEXT NOT NULL,
  type                TEXT CHECK (type IN ('brief', 'platform', 'campaign', 'site', 'other')),
  storage_path        TEXT,         -- null pour notes et liens
  content             TEXT,         -- note libre (DOC-5)
  is_pinned           BOOLEAN DEFAULT false,
  pinned_from_project UUID REFERENCES projects(id) ON DELETE SET NULL,
  owner_id            UUID REFERENCES auth.users(id) NOT NULL,
  created_at          TIMESTAMPTZ DEFAULT now()
);
```

<!-- Patterns Storage depuis la recherche -->
```typescript
// Signed upload URL (une seule utilisation)
supabase.storage.from('documents').createSignedUploadUrl(path)
// → { data: { signedUrl, path }, error }
// Path convention: `${user.id}/${clientId}/${Date.now()}-${filename}`

// Signed read URL (TTL 1h)
supabase.storage.from('documents').createSignedUrl(storagePath, 3600)
// → { data: { signedUrl }, error }

// Supprimer un fichier Storage
supabase.storage.from('documents').remove([storagePath])
```

```typescript
// createNote — INSERT document de type texte libre
// storage_path = null, content = le texte
// revalidatePath: '/[clientId]' page + '/[clientId]/[projectId]' page si projectId fourni
export async function createNote(params: {
  clientId: string
  projectId?: string
  name: string
  type: Document['type']
  content: string
}): Promise<{ error: string | null }>

// createLink — INSERT document de type lien externe
// storage_path = null, content = l'URL du lien
// revalidatePath: idem createNote
export async function createLink(params: {
  clientId: string
  projectId?: string
  name: string
  url: string
}): Promise<{ error: string | null }>

// createSignedUploadUrl — génère une signed URL d'upload (usage unique)
// Path: `${user.id}/${clientId}/${Date.now()}-${filename}`
// NE crée PAS de ligne en base — c'est uniquement l'étape 1 du 2-step upload
// Retourne { signedUrl, path } au succès
export async function createSignedUploadUrl(
  clientId: string,
  filename: string
): Promise<{ signedUrl: string; path: string } | { error: string }>

// saveDocumentRecord — étape 2 du 2-step upload
// INSERT avec storage_path = le path retourné par createSignedUploadUrl
// revalidatePath: '/[clientId]' page + '/[clientId]/[projectId]' page si projectId fourni
export async function saveDocumentRecord(params: {
  clientId: string
  projectId?: string
  name: string
  type: Document['type']
  storagePath: string
}): Promise<{ error: string | null }>

// getDocumentSignedUrl — génère une signed URL de lecture (TTL 1h = 3600s)
// Utilisé par DocumentViewer pour afficher un PDF
// Pas de revalidatePath — lecture seule
export async function getDocumentSignedUrl(
  storagePath: string
): Promise<{ signedUrl: string } | { error: string }>

// deleteDocument — DELETE row + supprime Storage si storage_path est set
// Fetch storage_path + client_id + project_id AVANT suppression
// Supprimer Storage AVANT la row DB pour éviter les orphelins
// revalidatePath: '/[clientId]' page + '/[clientId]/[projectId]' page si project_id présent
export async function deleteDocument(documentId: string): Promise<{ error: string | null }>

// pinDocument — UPDATE is_pinned=true + pinned_from_project=fromProjectId
// .eq('id', documentId).eq('owner_id', user.id)
// revalidatePath: '/[clientId]' page (le doc devient visible au niveau client)
export async function pinDocument(
  documentId: string,
  fromProjectId: string
): Promise<{ error: string | null }>
```

Pour `deleteDocument`, pattern séquentiel:
1. Fetch la row (`storage_path`, `client_id`, `project_id`) avec `.single()`
2. Si `doc.storage_path` existe: `await supabase.storage.from('documents').remove([doc.storage_path])`
3. DELETE la row
4. `revalidatePath()` sur les paths appropriés

Pour `createNote` et `createLink` avec `projectId` optionnel: appeler `revalidatePath('/[clientId]', 'page')` toujours + `revalidatePath('/[clientId]/[projectId]', 'page')` uniquement si `params.projectId` est défini.

Mettre à jour `DocumentViewer.tsx` pour supporter les PDFs uploadés dans Supabase Storage.

**Ce que fait le composant actuellement:** Il affiche `doc.content` (texte libre) dans une zone de texte.

**Ce qu'il doit faire après modification:**
- Si `doc.storage_path` est défini (PDF uploadé), appeler `getDocumentSignedUrl(doc.storage_path)` au montage du composant pour obtenir une URL signée, puis afficher le PDF dans un `<iframe>` ou `<embed>` (préférer `<iframe>` pour compatibilité maximale).
- Si `doc.storage_path` n'est pas défini (note ou lien), conserver l'affichage existant.

**Implémentation:**
1. Ajouter un `useState<string | null>` pour `pdfUrl`
2. Ajouter un `useEffect` qui: si `doc.storage_path` est défini → appelle `getDocumentSignedUrl(doc.storage_path)` → set `pdfUrl` sur succès
3. Dans le rendu: si `pdfUrl` → afficher `<iframe src={pdfUrl} className="w-full h-full min-h-[600px]" title={doc.name} />`; sinon, afficher le contenu existant

**Import:** `import { getDocumentSignedUrl } from '@/app/(dashboard)/actions/documents'`

Ne pas refactoriser l'UI existante au-delà de l'ajout du rendu PDF — pas de changement de layout, de styles, ni d'autres fonctionnalités.

Note: `doc.storage_path` n'existe pas sur le type `Document` de `mock.ts` (qui est le type actuel). Il peut être nécessaire d'ajouter un champ optionnel `storagePath?: string` au type Document, OU de caster `doc` via un type local étendu. Vérifier d'abord si `DocumentViewer` accepte un prop de type `Document` et adapter en conséquence. Préférer l'ajout d'un champ optionnel au type `Document` dans `mock.ts` pour éviter les casts.
