## Blocker 1 — AUTH-03: Route Protection Dead Code

**Severity:** BLOCKER
**File:** `proxy.ts` (root)
**Root cause:** Next.js App Router discovers middleware **exclusively by filename** — the file must be `middleware.ts` (root or `src/`). `proxy.ts` is never loaded by the framework.

**Impact:**
- `/dashboard` and `/projects/[id]` are accessible without authentication
- `updateSession()` / JWT refresh never fires on any request
- Only `/api/chat` and `/projects/[id]/chat` have inline `getUser()` guards

**Fix:** Rename `proxy.ts` → `middleware.ts` at the root.

---

## Blocker 2 — Missing `workflow_slug` in Documented Migration

**Severity:** BLOCKER (documentation / setup risk)
**File:** `.planning/phases/02-core-streaming/02-01-SUMMARY.md`
**Root cause:** The SQL migration that creates the `messages` table does not include the `workflow_slug` column added as a Phase 3 post-checkpoint fix.

**Impact:**
- Live DB is correct (column applied manually, user confirmed "done")
- Any fresh deployment from the documented migration will fail with a DB column error on first message send

**Fix:** Add `workflow_slug TEXT DEFAULT NULL` to the CREATE TABLE statement in 02-01-SUMMARY.md, or document the ALTER TABLE step.
