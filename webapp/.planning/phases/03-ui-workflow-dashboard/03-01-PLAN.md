**WORKFLOW_SYSTEM_PROMPTS** — per-workflow prompts. Each prompt instructs Claude to conduct the workflow as a conversational agent. Write these as complete, standalone operational instructions (the GBD workflows are conversational — multi-turn). Note: in the webapp context, the agent does NOT write files to disk. The output is produced in the chat conversation itself.

For each workflow, the system prompt should:
- Establish the agent persona (Tu es Brandon, ...)
- Describe the workflow's objective and the expected output artifact in the conversation
- List the workflow steps that must be followed in sequence
- Use French throughout

The 5 workflow prompts to write (inlined as TypeScript string template literals):

**"start"** — Brand strategy workflow. The agent:
  - Introduces itself and asks for the client brief
  - Proposes 3 strategic positioning angles based on the brief
  - Helps the user select and refine one angle
  - Co-constructs 5 strategic zones (Territoire, Valeurs, Personnalité, Promesse, Archétype) with the user
  - Concludes by presenting the complete CONTRE-BRIEF as a structured Markdown summary in the conversation
  - Announces "CONTRE-BRIEF CRÉÉ ✓ — Vous pouvez maintenant lancer /platform"

**"platform"** — Brand platform workflow. The agent:
  - Confirms it has received the contre-brief (asks user to paste it or confirm it exists in context)
  - Produces a 10-section brand platform document: Positionnement, Valeurs, Personnalité, Promesse, Territoire, Ligne éditoriale, Charte tonale, Mots-clés, Anti-exemples, Manifeste
  - Each section is 1-3 paragraphs
  - Concludes with "PLATFORM CRÉÉE ✓ — Vous pouvez maintenant lancer /campaign"

**"campaign"** — Creative campaign concept workflow. The agent:
  - Confirms it has received the brand platform
  - Generates a creative campaign concept: Big Idea, Axe créatif, Territoire visuel, Exemples de contenus (3-5), Hashtags, Call to action
  - Concludes with "CAMPAGNE CRÉÉE ✓"

**"site-standalone"** — Standalone site brief workflow. The agent:
  - Introduces itself as a site project director
  - Asks for the client's business context and site objectives (no brand platform required)
  - Co-constructs the site brief: Objectif principal, Cibles, Structure de navigation (5-7 pages), Ton et style, Références visuelles, Fonctionnalités clés
  - Concludes with "BRIEF SITE CRÉÉ ✓ — Vous pouvez maintenant lancer /wireframe"

**"wireframe"** — HTML wireframe workflow. The agent:
  - Confirms it has the site brief
  - Produces a complete HTML wireframe for the site (single HTML file content, inline CSS, no external dependencies)
  - The wireframe shows the layout, navigation, section headings, and placeholder content for each page
  - Outputs the full HTML in a code block
  - Concludes with "WIREFRAME CRÉÉ ✓"
