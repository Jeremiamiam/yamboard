    Export `WikiSection` accepting:
    - `id: string` — the section anchor id (value: `wiki-{slug}`)
    - `slug: string`
    - `label: string` — human-readable workflow name (e.g. "Contre-brief")
    - `outputJson: Record<string, unknown>`
    - `createdAt: string`

    - Looks for `outputJson.summary`, `outputJson.titre`, `outputJson.title`, `outputJson.big_idea`, `outputJson.positionnement` (first match wins, in that order)
    - If none found: takes the first string value of the first key in outputJson
    - If still none: returns "—"

    Render:
    - `<section id={id} ...>` with `className="mb-12 scroll-mt-8"` (scroll-mt-8 accounts for sticky header offset)
    - `<h2>` with `label` + formatted date (`new Date(createdAt).toLocaleDateString("fr-FR", { day: "numeric", month: "long", year: "numeric" })`)
    - TL;DR block: `<div className="bg-blue-50 border border-blue-200 rounded-lg p-4 mb-4">` with label "TL;DR" in `text-xs font-semibold text-blue-600 uppercase tracking-wide mb-1` and the extracted string in `text-sm text-blue-900`
    - Content block: render each key-value pair of `outputJson` as a section. Use `Object.entries(outputJson)`. For each entry: key as `<h3 className="text-sm font-semibold text-gray-700 mt-4 mb-1 capitalize">{key.replace(/_/g, " ")}</h3>`, value as `<div className="text-sm text-gray-600 whitespace-pre-wrap">{typeof value === "string" ? value : JSON.stringify(value, null, 2)}</div>`.

    No imports from external libraries beyond React (which is implicit in Next.js).

  <action>
    **`src/app/projects/[id]/wiki/page.tsx`** — Server Component (no "use client").

    Runtime: `export const dynamic = "force-dynamic"` — NOT `export const runtime = "edge"`. Use `nodejs` runtime (Supabase server client requires it). No explicit runtime export needed since nodejs is the default; just add `force-dynamic`.

    Wiki sections to display (in order): start → "Contre-brief", platform → "Plateforme", campaign → "Campagne", site → "Site", "site-standalone" → "Site". Filter to only slugs in this list.

    Define constant:
    ```typescript
    const WIKI_SECTIONS: Array<{ slug: string; label: string }> = [
      { slug: "start", label: "Contre-brief" },
      { slug: "platform", label: "Plateforme" },
      { slug: "campaign", label: "Campagne" },
      { slug: "site", label: "Site" },
      { slug: "site-standalone", label: "Site" },
    ]
    ```

    Fetch logic (same pattern as `src/app/projects/[id]/page.tsx`):
    1. `const supabase = await createClient()`
    2. Fetch project: `projects` table, select `id, name`, `.eq("id", id).single()` — call `notFound()` if null
    3. Fetch outputs: `workflow_outputs` table, select `id, workflow_slug, output_json, created_at`, `.eq("project_id", id)`, `.order("created_at", { ascending: false })`

    Build `sections` array:
    - For each entry in `WIKI_SECTIONS` (in order), check if `outputs` contains an entry with matching `workflow_slug`
    - If yes: include it with the output data
    - De-duplicate by slug: if both `site` and `site-standalone` have outputs, show the one that was completed most recently (just take first match in outputs array since ordered desc by date)

    Page layout:
    ```tsx
    <div className="min-h-screen bg-gray-50">
      <header className="bg-white border-b border-gray-200 px-6 py-4 flex items-center gap-4">
        <Link href={`/projects/${id}`} className="text-sm text-gray-500 hover:text-gray-900 transition-colors">
          ← {project.name}
        </Link>
        <h1 className="text-lg font-semibold">Wiki</h1>
      </header>
      <div className="max-w-5xl mx-auto px-6 py-8 flex gap-8">
        <aside className="hidden lg:block">
          <WikiSidebar sections={sidebarSections} />
        </aside>
        <main className="flex-1 min-w-0">
          {sections.length === 0 ? (
            <p className="text-gray-500 text-sm">Aucun workflow complété — revenez après avoir terminé au moins un workflow.</p>
          ) : (
            sections.map((s) => (
              <WikiSection
                key={s.slug}
                id={`wiki-${s.slug}`}
                slug={s.slug}
                label={s.label}
                outputJson={s.output.output_json}
                createdAt={s.output.created_at}
              />
            ))
          )}
        </main>
      </div>
    </div>
    ```

    `sidebarSections` = sections array mapped to `{ slug, label }`.

    Import: `createClient` from `@/lib/supabase/server`, `WikiSidebar` from `@/components/wiki-sidebar`, `WikiSection` from `@/components/wiki-section`, `Link` from `next/link`, `notFound` from `next/navigation`.
    Params pattern: `interface WikiPageProps { params: Promise<{ id: string }> }` then `const { id } = await params` — same as existing project page.
