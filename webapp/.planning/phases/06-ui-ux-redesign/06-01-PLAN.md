  <files>src/app/globals.css</files>
  <action>
Replace the current minimal globals.css with a full design token set using Tailwind v4's `@theme inline` block.

The app uses Tailwind v4 (not v3) — tokens are defined as CSS custom properties inside `@theme inline {}`, which Tailwind then exposes as utility classes.

Define the following token groups:

**Color palette (neutral professional, no branding):**
- `--color-bg: #F9FAFB` — page background (current gray-50)
- `--color-surface: #FFFFFF` — card/panel surface
- `--color-border: #E5E7EB` — borders (current gray-200)
- `--color-border-subtle: #F3F4F6` — subtle dividers (current gray-100)
- `--color-text-primary: #111827` — headings and strong text (current gray-900)
- `--color-text-secondary: #6B7280` — secondary/muted text (current gray-500)
- `--color-text-disabled: #9CA3AF` — disabled/placeholder text (current gray-400)
- `--color-brand: #2563EB` — primary action blue (current blue-600)
- `--color-brand-hover: #1D4ED8` — brand hover state (current blue-700)
- `--color-brand-subtle: #EFF6FF` — light blue background for brand highlights (blue-50)
- `--color-success: #16A34A` — success green (current green-600)
- `--color-success-subtle: #F0FDF4` — light green background (current green-50)
- `--color-success-text: #15803D` — success text (current green-700)
- `--color-danger: #DC2626` — error/danger red (current red-600)
- `--color-danger-subtle: #FEF2F2` — light red background (current red-50)
- `--color-danger-border: #FECACA` — red border (current red-200)

**Border radius:**
- `--radius-sm: 0.375rem` — 6px, for small elements like badges
- `--radius-md: 0.5rem` — 8px, for inputs, buttons
- `--radius-lg: 0.75rem` — 12px, for cards/panels
- `--radius-xl: 1rem` — 16px, for modals
- `--radius-2xl: 1.5rem` — 24px, for chat bubbles

**Keep existing:**
- `@import "tailwindcss"` at the top
- `@plugin "@tailwindcss/typography"`
- The `--font-sans` and `--font-mono` references to Geist font variables

**Remove:** The dark mode `@media (prefers-color-scheme: dark)` block — the app is light-only (internal tool, no dark mode per REQUIREMENTS.md Out of Scope).

**Body styles:** Set `background: var(--color-bg)` and `color: var(--color-text-primary)` on body, replacing the current CSS custom property indirection.

Final structure:
```css
@import "tailwindcss";
@plugin "@tailwindcss/typography";

@theme inline {
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);

  /* Colors */
  --color-bg: #F9FAFB;
  ... (all tokens above)

  /* Border radius */
  --radius-sm: 0.375rem;
  ... (all radius tokens)
}

body {
  background: var(--color-bg);
  color: var(--color-text-primary);
  font-family: var(--font-geist-sans), Arial, sans-serif;
}
```
